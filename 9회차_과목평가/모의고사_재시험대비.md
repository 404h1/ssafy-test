## 문제 07 정답 (5점)

**수정 파일:** `accounts/views.py`

```python
from django.contrib.auth import login as auth_login

def signup(request):
    if request.method == 'POST':
        form = CustomUserCreationForm(request.POST)
        if form.is_valid():
            user = form.save()
            auth_login(request, user)   # 자동 로그인 추가
            return redirect('todos:index')
    else:
        form = CustomUserCreationForm()
    context = {'form': form}
    return render(request, 'accounts/signup.html', context)
```

**핵심 개념:** `auth_login(request, user)` — 세션에 사용자 정보를 저장하여 로그인 처리

# 빠른 체크리스트

시험 직전 반드시 확인할 것:

## Django 자주 나오는 버그 패턴

| 증상 | 원인 | 수정 |
|------|------|------|
| 비로그인 사용자 접근 허용 | `@login_required` 누락 | 뷰 함수 위에 데코레이터 추가 |
| 수정 폼에 기존 데이터 없음 | `instance=` 누락 | `Form(instance=obj)` |
| 저장 시 IntegrityError | FK 필드 할당 누락 | `commit=False` 후 `obj.user = request.user` |
| 이미지 저장 안됨 | `enctype` 누락 | `enctype="multipart/form-data"` |
| 회원가입 후 로그아웃 상태 | `auth_login` 누락 | `auth_login(request, user)` |
| NoReverseMatch | URL 인자 타입 불일치 | `user.pk` vs `user.username` 확인 |
| Forbidden 403 | CSRF 토큰 누락 | `{% csrf_token %}` 추가 |
| 삭제/변경이 안됨 | form method GET | `method="POST"` 로 수정 |
| 목록 정렬 안됨 | `order_by` 누락 | `.order_by('-created_at')` |

## SQL 자주 나오는 패턴

```sql
-- 기본 필터 + 정렬
SELECT Name, Price FROM Product WHERE Price >= 50 ORDER BY Price DESC;

-- 집계 (GROUP BY)
SELECT CategoryId, COUNT(*) AS cnt, MAX(Price) AS max_price
FROM Product WHERE StockQuantity > 0
GROUP BY CategoryId ORDER BY max_price DESC;

-- 조인
SELECT DISTINCT u.Username, u.Email
FROM User u JOIN Purchase p ON u.Id = p.UserId
WHERE p.Status = 'completed' ORDER BY u.Username;

-- 조인 + 집계
SELECT p.Name, COUNT(pi.Id) AS purchase_count
FROM Product p JOIN PurchaseItem pi ON p.Id = pi.ProductId
GROUP BY p.Id ORDER BY purchase_count DESC;

-- 사용자별 합계
SELECT u.Username, SUM(p.TotalPrice) AS total_spent
FROM User u JOIN Purchase p ON u.Id = p.UserId
WHERE p.Status = 'completed'
GROUP BY u.Id ORDER BY total_spent DESC;
```
