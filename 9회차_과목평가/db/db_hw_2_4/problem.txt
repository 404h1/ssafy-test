학습주제
시스템을 위한 데이터베이스 설계와 구축
SQL 쿼리 작성 및 활용

학습목표
데이터베이스 스키마를 구성하는 방법과 데이터베이스 테이블을 생성하는 방법을 이해한다.
데이터를 삽입, 수정, 삭제하는 방법을 숙지한다.
데이터베이스에서 SELET 문을 사용하여 데이터를 조회하는 방법을 습득한다.

문제의도
시스템 설계를 위한 데이터베이스 설계와 SQL 쿼리 작성, 데이터베이스 조작 방법에 대해 이해한다.

문제
키오스크 제작 업체에서 주문 정보를 관리 할 수 있는 데이터베이스를 구축해달라는 요청을 받았다. 요청사항은 orders 테이블의 구조가 잘못 설계 되어 있어 이를 삭제하고, 새로 만들어야 하는 것이었다. 그런데 문제는, 최종 형태가 될 데이터 베이스의 설계도가 분실되어서 도중의 설계도를 토대로 우선 테이블을 생성하고 필드를 수정하는 과정을 거쳐야 한다는 것이다. 제공된 이미지를 참고하여 요구 사항을 만족하는 데이터베이스를 구축하시오.
요구사항
data.sql 파일을 생성한다.
SQL문은 생성한 data.sql 파일에서 작성하고 실행한다.
실행 방법
각 요구사항에 적절한 SQL문을 작성한다.
실행하고자 하는 SQL문을 우클릭한다.
Run Selected Query 를 선택하여 실행한다.
결과를 확인한다.

orders 테이블을 삭제한다.
orders 테이블을 첨부 파일을 참고하여 다시 생성한다. customer와 관계를 형성한다.

쿼리문을 사용하여 orders 테이블을 수정한다 
orders 테이블에 Integer 타입의 price 컬럼 추가한다. 
orders 테이블의 total_amount 컬럼을 삭제한다.

orders 테이블에 데이터를 최소 3개 이상 생성한다. 
Date 타입은 'YYYY-MM-DD' 형식으로 삽입한다. 
Real 타입은 10.11 형식으로 삽입한다. 
데이터를 생성할 때 customers_id도 함께 삽입한다.

orders의 모든 데이터를 조회한다. 단, 관계를 맺고 있는 customers의 name도 함께 출력한다.
